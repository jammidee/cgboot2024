Im developing an API that will allow me to:
1. setup an ollama enpoint, query for available LLMs, etc.
2. setup what type of LLM to download, using ollama
3. ability to call an LLM via API by sending a prompt
4. Ablility to keep a persona per api all so that I can continue the LLM memory across inqueries. Then persona is used again
   if the same persona is passed as parameter. The persona lives in a mysql database. A persona is clear when a reset API is called
5. Create an API to meet these requirements. Use routes/api/mc for this.

The goal is to network different LLMs laters and draw summary on their final result.

const bcrypt  = require('bcrypt')
const User    = require('../../models/user.model')

/**
 * LoginService
 * Handles authentication logic only.
 * No HTTP logic here.
 */
class AuthService {

  /**
   * Validate username & password
   */
  async authenticate(email, password) {

    const user = await User.findOne({
      where: { email }
    })

    if (!user) {
      throw new Error('Invalid username or password')
    }

    const match = await bcrypt.compare(password, user.password)

    if (!match) {
      throw new Error('Invalid username or password')
    }

    return user
  }
}

module.exports = AuthService
